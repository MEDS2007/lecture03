# CSS RULES **1** the **selector** is the element to be styled: ```css selector { /* can be element, class… */ property: value; } ``` inside the curly braces `{ }`, the **property** and **value** together make a CSS **declaration** e.g. ```css main { margin: 0 auto; } .sidebar { width: 50%; } .contact-page form { /* the form on the contact page */ background: #666; } ```
# CSS RULES **2** A CSS **rule** inside `{}` can contain **multiple** CSS **declarations**: ```css main { margin: 10px; /* 10 pixels all round */ width: 80%; /* always 80% of the screen width */ background: #eee; font-family: Tahoma, Helvetica, sans-serif; } ``` or be on **one line** (*harder to read* when developing): ```css main { margin: 0 auto; width: 50%; } ```
# CSS RULES **3** There are several **selector types**—they can also be: - [pseudo-classes](https://developer.mozilla.org/en-US/docs/Web/CSS/Pseudo-classes) (e.g. `:hover` or `:checked`) - [pseudo-elements](https://developer.mozilla.org/en-US/docs/Web/CSS/Pseudo-elements) (e.g. `::before` or `::first-line`) CSS **selector references**: - [List of selectors](http://www.w3schools.com/cssref/css_selectors.asp) (W3Schools) - [Interactive page of CSS selectors](http://www.w3schools.com/cssref/trysel.asp) (W3Schools)
# BOX MODEL **1** All **HTML tags** create a box or `block`: ![A diagram of the box model](images/boxmodel.png)
# BOX MODEL **2** The **dimensions** of the box are controlled by the: - **margin:** space ***outside*** - **padding:** space ***inside*** - **border:** line ***around*** *inline* HTML elements (e.g. `a` or `span` tags) **are also boxes**, but to make their **margin or padding** visible you need CSS rules that turn them into **block elements**: `display: block;` for a line-break before and after or `display: inline-block;` to keep them inline, e.g. links in text
# BOX MODEL **3**: MARGIN The **margin** makes an invisible **space around** an element It is usually set in **pixels**, **ems** or **percentages**: `margin: 6px;` (or `.5em` or `5%`…) Or **no margin** with `margin: 0;` which removes any **default margins** that browsers apply to the following elements: `body`, `h1` to `h6`, `p`, `blockquote`, `form`, `fieldset`, and the list elements: `ul`, `ol`, `dl`, `dd`
# BOX MODEL **4**: PADDING **Padding** is **around** the box **content** but still **inside** the box, padding includes the **background colour** and **background image** of a box usually set in **pixels**, **ems** or **percentages**: `padding: 4px;` (or `1em` or `5%`…) Can be zero: `padding: 0;`
# BOX MODEL **5**: BORDER The **border** goes **around the outside** of the box best set in **pixels**, and easiest to set in one declaration: `border: 1px solid silver;` `border-bottom: 3px dotted #999;` Can be zero: `border-top: none;`
# BOX MODEL **6**: 3D If you could **see HTML boxes**, they'd look like this! Note the **layers** are **nested** just as in the **HTML** ![3D box model image](images/3dview.png)
# BOX-MODEL **SYNTAX**: 1 By default, **padding and border** are **added** to the width and height of a box: ```css main { width: 70%; padding: 1em; /* 1em; the height of one line of text */ border: 2px; } ``` width is 70% **+** 2em **+** 4px (very hard to work with)
# BOX-MODEL **SYNTAX**: 2 With `box-sizing` set to to `border-box` the **padding and border** are **included** in the width and height, making measurements easier: ```css main { box-sizing: border-box; width: 70%; padding: 1em; border: 2px; } ``` `border-box` means the width **remains 70%** (*not* 70% + 2em + 4px)
# BOX MODEL **SYNTAX**: 3 ## For both **margin** and **padding** ```css header { margin: 2px; /* a 2-pixel margin all round */ padding-top: 1em; /* one line of padding at top */ margin-left: 5%; /* 5% of available space to left */ } /* you can use: -top -right -bottom -left */ ```
# BOX-MODEL **SYNTAX**: 4 **CSS shorthand** - **4** values: **top, right, bottom, left** (clockface) - **3** values: **top, left-right, bottom** - **2** values: **top-bottom, left-right** ```css header { padding: 1em 10px .5em 5px; /* 4: top right bottom left: */ padding: 1em 10px .5em; /* 3: top 1em, l-r 10 pixels, bottom .5em */ margin: 1em 0; /* 2: one line top+bottom, none left-right: */ } ```
# BOX-MODEL **SYNTAX**: 5 You can set top, right, bottom, left, then **reset one** ```css header { margin: 10px; /* 10 pixels all round… */ margin-top: 0; /* …but no top margin */ } ``` **IMPORTANT**: The second line comes *after*—so `margin-top` **overrides** any rule above it. The other margins remain at `10px`
# BOX-MODEL **SYNTAX**: 6 To **centre** an block element by itself, set the **left** and **right** margins to `auto`: ```css header { margin: 0 auto; /* zero top-bottom, auto l-r */ } ``` `auto` fills the available space to the left and right equally. This is also a common way to **centre a page design** in the **browser window**, although many current designs prefer to **fill the browser window** completely.
# BOX-MODEL **SYNTAX**: 7 `display: inline-block` turns **inline elements** into **blocks that line up horizontally**. For instance, with `a` or `li` tags in menus (inside `nav` tags): ```css nav a { /* a tags inside a nav tag */ display: inline-block; } ``` in the rule above, 'block' makes inline `a` elements into blocks, so they can show `padding` and `margin` rules. [W3Schools: HTML Block and Inline Elements](http://www.w3schools.com/html/html_blocks.asp)

# BOX MODEL **BORDER**: 1 box outline with three parameters: `width`, `style`, `color` ```css .mybox img { /* separately: */ border-width: 2px; border-style: solid; /* or: none|solid|dotted|dashed */ border-color: white; } .mybox img { /* shorthand: */ border: 2px solid white; /* width/style/color */ } ``` …if the image is inside a **figure** tag with a **class "mybox"**: ``` <figure class="mybox"> <img src="images/mypic.png" alt="my face"> </figure> ```
# BOX-MODEL **BORDER**: 2 Each property can have **one**, **two**, **three** or all **four** “clockface” values: **top**, **right**, **bottom**, **left**. ```css header { border-color: white black; /* top, bottom */ border-width: 2px 0 4px; /* top, r-l, bottom */ border-style: solid dotted none dashed; /* top, right, bottom, left */ } ```
# BOX-MODEL **BORDER**: 3 or you can use **CSS shorthand** to set a single border style all around an element: ```css img { border: 1px #555 dotted; /* a grey dotted border */ } ```
# BOX-MODEL **BORDER**: 4 Boxes can be rounded with `border-radius`: ```css aside { border-radius: 2px; /* slight corner curves */ } img { border-radius: 50%; /* totally round or oval */ } ```
# BOX-MODEL **BORDER**: 5 Or have one, two or three sides rounded… ```css img { border-top-left-radius: 50%; border-bottom-left-radius: 50%; /* a semi-circle (if height=width) or half-ellipse */ } img { border-top-right-radius: 100%; /* an arc (if height=width) */ } ``` Here are some examples…

# FLEXBOX **1**: INTRODUCTION **CSS flexbox** is for **arranging boxes** inside a container. For example, for a container that has a class `my-boxes`, first **apply the flexbox rule** to the **container element**: ```css .my-boxes { display: flex; } ``` The **default** style is to arrange the boxes in a horizontal row, but other rules can be added to **arrange them differently**…
# FLEXBOX **2**: HORIZONTAL for **horizontal alignment** use the `justify-content:` property, with values: - `flex-start;` group items to **left** (the start) of container - `flex-end;` group items to **right** of container - `center;` group items in the **center** of a container - `space-between;` **distribute** items: first/last are left/right others are equally between - `space-around;` **distribute** and space items **evenly**
# FLEXBOX **3**: VERTICAL for **vertical alignment** use the `align-items:` property, with values: - `flex-start;` across top - `flex-end;` across bottom - `center;` in center - `baseline;` across baseline - `stretch;` space items, or stretch item to fill
# FLEXBOX **4**: ORDER for the **order of boxes** use the `flex-direction:` property, with values: - `row;` flow left to right - `row-reverse;` flow right to left - `column;` stack top to bottom - `column-reverse;` stack bottom to top
# FLEXBOX **5**: EXAMPLE In this example, the **parent element** `.my-boxes` contains several **child elements** to arrange: ```css .my-boxes { display: flex; /* set to display as flexbox */ justify-content: space-around; /* horizontal spacing */ align-items: center; /* align vertical center */ flex-direction: row-reverse; /* wrap last-first */ } ```
# FLEXBOX **6**: EXAMPLE The HTML just needs a **selector** e.g. `class="my-boxes"` for the CSS to **identify the element** to be **styled as flexbox**: ```html <section class="my-boxes"> <div>Box one</div> <div>Box two</div> <div>Box three</div> </section> ```
